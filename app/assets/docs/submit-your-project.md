# Submit Your Project

When your build is ready, prepare a submission package that reviewers can verify quickly.

## Submission checklist

1. Repository is public and readable.
2. Journal or timeline shows steady progress.
3. Bill of materials is complete.
4. Build instructions are reproducible.
5. Demo media clearly shows the final behavior.

## Recommended structure

- `README.md` with overview and setup
- `Journal.md` for daily logs
- `firmware/` and `hardware/` folders
- `images/` folder for final photos

## Final pass

- Re-test from a clean setup
- Fix broken links and missing files
- Add short captions for every image
- Tag a release before submission

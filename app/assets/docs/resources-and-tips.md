# Resources and Tips

Use trusted references and keep your workflow repeatable.

## Useful resources

- [Raspberry Pi Documentation](https://www.raspberrypi.com/documentation/)
- [MicroPython Docs](https://docs.micropython.org/)
- [CircuitPython Docs](https://docs.circuitpython.org/)
- [KiCad Documentation](https://docs.kicad.org/)

## Debugging tips

- Test hardware and firmware separately
- Keep known-good test scripts
- Use clear commit messages for every checkpoint
- Capture photos when something changes physically

## Writing better docs

- Start with a one-minute overview
- Include a parts list with exact quantities
- Add wiring diagrams and assumptions
- Include common failure modes and fixes
